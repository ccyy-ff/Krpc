# 2、技术栈需求以及项目环境

# c++基础

掌握面向对象编程：三大特性封装、继承、多态。

虚函数：掌握虚函数的使用最起码要理解什么是多态。

掌握基本stl的使用:常用的容器如(vecotr<T>、undordered\_map，set等)

设计模式：单例模式(具体在Krpcapplication.h)

模板：最好掌握模板的使用

# 网络编程

* **Socket 编程**\
  理解 TCP 和 UDP 的工作原理，熟悉 Socket API 的使用（`socket()`, `bind()`, `listen()`, `accept()` 等）。
* **异步与非阻塞 I/O**\
  掌握常见的 I/O 模型（阻塞、非阻塞、IO 多路复用、异步 I/O），熟悉 `epoll`, `select`, `poll` 等机制。
* **网络协议**\
  理解 HTTP、gRPC 等高层协议，以及基于 TCP/UDP 的自定义协议设计。

# 数据序列化与反序列化

* **序列化格式**\
  学习常用的序列化格式，例如：
  * **Protobuf**: 高效、跨语言支持广泛。
  * **Thrift**: 集成服务定义和序列化。
  * **MessagePack / JSON / XML**: 轻量级或人类可读的格式。
  * **自定义协议**\
    如果需要实现特定场景的优化，需学习如何设计紧凑的二进制数据协议。
* 推荐学习：\
  [protobu官方文档](https://protobuf.com.cn/programming-guides/proto3/)：本项目主要使用的是proto3

# RPC基础

**概念**

1. <font style="color:rgb(47, 48, 52);">RPC（Remote Procedure Call Protocol） 远程过程调用协议。</font>
2. <font style="color:rgb(47, 48, 52);">RPC是一种通过网络从远程计算机程序上请求服务，不需要了解底层网络技术的协议。</font>
3. <font style="color:rgb(47, 48, 52);">RPC主要作用就是不同的服务间方法调用就像本地调用一样便捷。</font>

主要的内容会在下面的模块进行详细介绍
